Ниспадающий список¶

С помощью компонента можно создавать ниспадающие списки с ручным вводом значений.

Как задать компонент формы?¶

"objectType": {
                "type": "select",
                "title": "Выберите типа объекта",
                "enum": [ "Жилой комплекс", "Спортивные сооружения", "Служебные помещения" ]
}

  1. workers-amount — это FIELDNAME. Задётся пользователем и необходимо для объявления любого компонента.

  2. type — тип элемента. Для ниспадающего списка — это всегда select.

  3. title — название поля, отображаемое в форме. Формат: string.

  4. enum — список доступных значений. Если значений больше одного, они вписываются в квадратных скобках — [] Формат: string.

Пример¶

JSON-схема для компонента в конфигураторе:

"objectType": {
                "type": "select",
                "title": "Выберите типа объекта",
                "enum": [ "Жилой комплекс", "Спортивные сооружения", "Служебные помещения" ]
}

Как сохраняются данные?¶

Данные сохраняются в FormInstances в конфигураторе.

После сохранения данные попадают в FromInstances в конфигураторе. Они сохраняются также в JSON формате:

{
    "objectType": "Жилой комплекс"
}
