Справочники
===========

Справочник ー это именованная коллекция структурированных машиночитаемых сведений,
имеющих одинаковую структуру и моделирующих какую-то сущность из предметной области.

Пример: организации, сотрудники, виды работ, участки.

Запись справочника ー это единица информации из Справочника, соответствует структуре справочника,
обладает собственной идентичностью и может быть адресована по идентификатору.

В отличие от Форм Справочники не обладают собственной бизнес-логикой с точки зрения системы.
То есть для них нельзя задать хуки.

В то же время записи в Справочниках могут быть адресованы по идентификатору,
то есть на них можно ссылаться из Форм или других справочников. 

Справочники  могут использоваться для наполнения отчетов,
как классификаторы для объектов бизнес-логики (типы замечаний, разрезы шахматки и тд).
Модель данных справочников настраивается в зависимости от задачи, поставленных перед конфигурацией. 

Система позволяет создавать неограниченное количество справочников.
Справочники могут быть заполнены вручную или путем интеграции.

Создание справочников происходит в Directus. Далее вы можете узнать как создавать :doc:`типовые структуры справочников<dictionaries/dictionaries-types>`.
